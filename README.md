# Сравнение производительности: Чистый код (Python) vs No-Code платформа (Dify)

В рамках исследования архитектурных решений для автоматизации бизнес-процессов было проведено практическое сравнение двух подходов к реализации агентских цепочек (Multi-Agent Chains).

### Сценарий тестирования
Реализована идентичная цепочка из двух агентов для обработки входящих обращений:
1. Агент-Классификатор (Llama 3): Анализирует запрос пользователя и определяет категорию (`TECH`, `BILLING` или `SPAM`).
2. Агент-Решатель (Llama 3): На основе категории и сути запроса формирует итоговый вежливый ответ клиенту.

Стек и окружение:
   - Локальная LLM: Llama 3 (8B) запущенная через Ollama.
   - Решение 1 (No-Code): Локально развернутая платформа Dify (сборка через визуальный редактор Workflow).
   - Решение 2 (Code): Скрипт на Python с асинхронными HTTP-запросами напрямую к API Ollama.
   - Параметры теста: 35 последовательных итераций запроса "У меня не списывается оплата за подписку, помогите!".

---

## Результаты бенчмарка (35 итераций)

| Метрика | Чистый код (Python + Ollama) | No-Code платформа (Dify) |
| :--- | :---: | :---: |
| Среднее время выполнения запроса | 81.37 сек | 56.33 сек  |
| Всего потрачено токенов (среднее) | 294 токенов | 105 токенов |
| Сгенерировано токенов (среднее out) | 143 токенов | 0 токенов |
| Реальная скорость генерации (TPS) | 1.8 tok/sec | 0.0 tok/sec |

Примечание: Более низкое время выполнения у Dify обусловлено тем, что платформа не отдавала сгенерированный текст во втором блоке (вернула пустой ответ / 0 токенов), из-за чего завершала шаг мгновенно. Чистый код успешно выполнил полную цепочку рассуждений и генерации.

## 🧠 Ключевые выводы исследования

Рекомендации по выбору стека для проекта:

   - Dify (No-Code) идеально подходит на этапе PoC (Proof of Concept) и быстрого прототипирования: позволяет нетехническим специалистам оперативно "набросать" логику и проверить гипотезу за 15 минут.

   - Чистый код (микросервисы) обязателен для Production-среды под нагрузкой. Он гарантирует стабильность выполнения цепочки, минимальные задержки (latency), полный контроль над потреблением токенов и предсказуемость логики.
